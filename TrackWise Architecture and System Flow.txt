---------------------------------------------------------------------------
COMPONENTS (FRONTEND UI DESIGN)
---------------------------------------------------------------------------

Button
    - add employee, add department, add task
    - update employee, update department, update task
    - delete employee, delete department, delete task

Form (Reusable per feature)
    - employee form (create + update)
    - department form (create + update)
    - task form (create + update)

    * uses FormInput and SelectDropdown from Shared

Table
    - display list of employees (employee info, department name)
    - display list of departments (department info)
    - display list of tasks (task name, employee name, priority level, status level)
    - display list of assigned tasks (task, employee)

Dropdown (Select Component)
    - priority level selection (Low, Medium, High)
    - status level selection (Not Started, In Progress, Completed)

Modal
    - used for create forms (employee, task, department)
    - used for update forms
    - used for delete confirmation dialogs

ARCHITECTURE OVERVIEW
---------------------------------------------------------------------------

Backend (.NET)
- Layered Architecture

Controller → Service → Repository → Database

Frontend (React)
- Component-Based Architecture

Page → Components → Services → UI Elements

---------------------------------------------------------------------------
COMPARISON TABLE
---------------------------------------------------------------------------

| Concept        | Backend (.NET)                | Frontend (React)           |
| -------------- | ----------------------------- | -------------------------- |
| Architecture   | Layered                       | Component-Based            |
| Structure      | Controller → Service → Repo   | Page → Components          |
| Principle used | SOLID                         | SRP (applied in components)|
| Goal           | Business + data processing    | UI composition             |

EXAMPLE FLOW: CREATE EMPLOYEE
---------------------------------------------------------------------------

User clicks Add Employee (React)
        ↓
Modal opens (EmployeeForm)
        ↓
Form submits data
        ↓
employeeService.createEmployee()
        ↓
POST /api/employee
        ↓
EmployeeController receives request
        ↓
CreateEmployeeRequest (DTO)
        ↓
EmployeeService validates business rules
        ↓
EmployeeRepository executes SQL INSERT
        ↓
DbConnectionFactory opens connection
        ↓
SQL Server stores data
        ↓
Response returned (EmployeeResponse)
        ↓
UI updates Employee Table
