ARCHITECTURE
----------------------------------------------------------------------------------------------------------
Backend (.NET)
- uses Layered Architecture
Controller → Service → Repository → Database

Frontend (React)
- uses Component-Based Architecture
Page → Components → UI Elements

	BACKEND			FRONTEND
-------------------------------------------------------
| Layered Architecture | Component-Based Architecture |
-------------------------------------------------------
| Strict layers	       | Flexible structure           |
-------------------------------------------------------
|Business rules in services| Logic in hooks/services  |
-------------------------------------------------------

| Concept        | Backend                | Frontend            |
| -------------- | ---------------------- | ------------------- |
| Architecture   | Layered                | Component-based     |
| Structure      | Layers                 | Components          |
| Principle used | SOLID                  | SRP (part of SOLID) |
| Goal           | Separation of concerns | Reusable UI pieces  |

--------------------------------------------------------------------------------------------------------
SYSTEM FLOW
--------------------------------------------------------------------------------------------------------

TRACKWISE SYSTEM FLOW (FULL ARCHITECTURE)

React UI (Frontend)
        ↓
API Service Layer (Axios / Services)
        ↓
ASP.NET Core Web API (Backend)
        ↓
Controller
        ↓
Service (Business Logic)
        ↓
Repository (Data Access)
        ↓
Database (SQL Server)



FRONTEND FLOW (Component-Based Architecture - React SPA)

Page
  ↓
Components
  ↓
Hooks (optional logic)
  ↓
Services (API calls)


BACKEND FLOW (Layered Architecture - SOLID)

Controller
   ↓
Service (Business Logic)
   ↓
Repository (Data Access / ADO.NET)
   ↓
Data Layer (CreateConnection)
   ↓
Database (SQL Server)
------------------------------------------------------------------
EXAMPLE: “Create Employee”

Controller receives request
   ↓
Calls EmployeeService.CreateEmployee()
   ↓
Service validates business rules
   ↓
Calls EmployeeRepository.InsertEmployee()
   ↓
Repository uses CreateConnection()
   ↓
Executes SQL INSERT
   ↓
Data saved in SQL Server
   ↓
Result returned back to Controller
   ↓
Controller returns API response to React
-----------------------------------------------------------------
User clicks "Add Department"
        ↓
Modal opens
        ↓
User submits form
        ↓
Hook → API → success
        ↓
Form resets + Modal closes
        ↓
Table updates automatically
